# DOKUMEN 19 — QUALITY GATE EDITOR TP FLAF
Versi 2.0

## BAB 1 — TUJUAN
Quality Gate Editor merupakan pemeriksaan akhir
dalam memori sebelum tp-xx-v6.js ditulis.

Quality Gate Editor bukan proses audit.
Quality Gate Editor bertujuan memverifikasi bahwa
seluruh perubahan telah dilakukan sesuai Audit OS
dan Spesifikasi Editor.

## BAB 2 — FILOSOFI
tp-xx-v6.js tidak dinyatakan siap karena terlihat lebih baik.
tp-xx-v6.js dinyatakan siap apabila seluruh perubahan:
* berasal dari Audit OS;
* mempertahankan tujuan pedagogis;
* tidak menambah perubahan yang tidak diperlukan;
* siap live di browser.

## BAB 3 — GATE A — Kepatuhan terhadap Audit OS
□ seluruh temuan telah diterapkan;
□ tidak ada temuan yang diabaikan tanpa alasan;
□ setiap perubahan dapat ditelusuri ke Audit OS.

## BAB 4 — GATE B — Integritas TP
□ tujuan pembelajaran tetap sama;
□ strategi pembelajaran tetap sama;
□ urutan aktivitas tetap sama;
□ level kesulitan tetap sama.

## BAB 5 — GATE C — Perubahan Minimum
□ tidak ada aktivitas baru;
□ tidak ada aktivitas yang dihapus;
□ tidak ada penjelasan baru;
□ tidak ada variasi yang tidak diminta.

## BAB 6 — GATE D — Konsistensi Revisi
□ istilah tetap konsisten;
□ format tetap konsisten;
□ UCAP tetap konsisten;
□ struktur TP tetap konsisten.

## BAB 7 — GATE E — Beban Kognitif
□ revisi mengurangi interpretasi;
□ revisi mengurangi keputusan tambahan;
□ revisi tidak menambah beban baca;
□ revisi tidak menambah beban proses.

## BAB 8 — GATE F — Ruang Lingkup Editor
□ editor tidak melakukan audit baru;
□ editor tidak membuat rekomendasi baru;
□ editor tidak melakukan redesign;
□ editor tidak melakukan improvement di luar Audit OS;
□ editor tidak membuat file perantara.

## BAB 9 — KEPUTUSAN

### SIAP DITULIS
Apabila seluruh Gate memenuhi standar.
→ Tulis tp-xx-v6.js
→ SW bump + commit + push

### BELUM SIAP
Apabila terdapat satu saja kondisi berikut:
* ada temuan tidak dapat diterapkan;
* ada perubahan yang mengubah tujuan pedagogis;
* ada perubahan di luar Audit OS.
→ Tampilkan ke owner
→ Berikan rekomendasi perbaikan
→ Tunggu keputusan owner + Claude (chat)

## BAB 10 — PERTANYAAN AKHIR
Sebelum tp-xx-v6.js ditulis, editor wajib menjawab:

Apakah seluruh perubahan pada TP ini
dapat dijelaskan dengan merujuk pada Audit OS,
dan apakah file ini siap live di browser?

Jika Ya → tulis file.
Jika Tidak → tampilkan ke owner.

## BAB 11 — PRINSIP NORMATIF
Quality Gate Editor adalah gerbang kepatuhan.

Standar keberhasilannya adalah:
* seluruh temuan audit telah diterapkan;
* tidak ada perubahan di luar Audit OS;
* TP tetap mempertahankan tujuan pedagogis;
* tp-xx-v6.js siap live di browser.
