# DOKUMEN 15 — FLAF EDITOR OPERATING SYSTEM
Versi 3.0

## TUJUAN
Editor OS menetapkan cara kerja standar Claude Code
untuk menghasilkan TP final berdasarkan Audit OS.

## LANGKAH PERTAMA DI SETIAP SESI
Baca:
1. Workflow (Dokumen 14)
2. tp-xx-v5.js

Tampilkan hanya:
* STATUS
* FILE TARGET
* NEXT TASK

Tunggu instruksi sebelum melakukan perubahan.

## PERAN EDITOR
Editor bukan auditor.
Editor tidak mencari kesalahan baru.
Editor hanya:
* menerapkan temuan audit;
* memperbaiki TP;
* menjaga tujuan pedagogis;
* menjaga konsistensi;
* mengurangi beban kognitif.

## SOURCE OF TRUTH
Urutan prioritas:
1. Audit OS
2. Workflow
3. Editor OS

## WORKFLOW
Baca tp-xx-v5.js
↓
Audit + Edit + Quality Gate (dalam memori)
↓
Jika semua beres
→ terapkan perubahan dengan str_replace
→ SW bump + commit + push
↓
Jika ada temuan tidak dapat diterapkan
→ tampilkan ke owner
→ berikan rekomendasi perbaikan
→ tunggu keputusan owner + Claude (chat)
↓
TULIS → terapkan perubahan dengan str_replace
        + SW bump + commit + push
PERBAIKI → revisi → tampilkan lagi → tunggu

## ATURAN WAJIB
Editor WAJIB:
* mempertahankan tujuan pembelajaran;
* mempertahankan strategi pedagogis;
* mempertahankan struktur aktivitas;
* mempertahankan urutan pembelajaran.

Editor DILARANG:
* membuat file baru;
* membuat folder baru;
* menemukan temuan baru saat fase edit;
* mengubah desain pembelajaran;
* menambah atau menghapus aktivitas;
* melakukan improvement di luar Audit OS.

## PRINSIP EDIT
Setiap revisi harus:
* menghilangkan masalah yang ditemukan auditor;
* tidak menciptakan masalah baru;
* mengurangi beban kognitif;
* mempertahankan konsistensi TP.

## OUTPUT
tp-xx-v5.js yang telah direvisi langsung
menggunakan str_replace.

## QUALITY CHECK INTERNAL
Sebelum selesai, editor wajib memeriksa:
□ seluruh temuan telah diterapkan;
□ tidak ada perubahan di luar temuan;
□ tujuan pedagogis tetap sama;
□ struktur TP tetap sama;
□ tidak muncul keputusan operasional baru;
□ tidak muncul beban kognitif baru.

## PRINSIP AKHIR
Keberhasilan Editor OS diukur dari satu pertanyaan:
Apakah seluruh temuan audit telah diterapkan
menggunakan str_replace tanpa menimbulkan temuan baru,
menghasilkan TP yang siap live di browser?
