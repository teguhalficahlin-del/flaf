# DOKUMEN 16 — SPESIFIKASI EDITOR TP FLAF
Versi 2.0

## BAB 1 — TUJUAN
Spesifikasi ini menetapkan standar Editor TP FLAF.

Editor bertugas menghasilkan tp-xx-v6.js
berdasarkan hasil Audit OS.

Editor tidak melakukan audit ulang.
Editor tidak menetapkan standar baru.
Editor tidak membuat file perantara.

## BAB 2 — FILOSOFI EDIT
Edit bukan proses memperbaiki menurut pendapat editor.
Edit adalah proses menerapkan keputusan Audit OS
secara konsisten dalam memori,
lalu menghasilkan satu file final.

Ukuran keberhasilan editor bukan banyaknya perubahan,
melainkan ketepatan perubahan.

## BAB 3 — PRINSIP DASAR

### 3.1 Prinsip Kepatuhan
Seluruh perubahan harus berasal dari Audit OS.
Editor tidak membuat perubahan di luar Audit OS.

### 3.2 Prinsip Perubahan Minimum
Apabila satu masalah dapat diperbaiki
dengan perubahan kecil,
maka perubahan besar tidak boleh dilakukan.

### 3.3 Prinsip Tujuan Tetap
Edit tidak boleh mengubah:
* tujuan pembelajaran;
* strategi pedagogis;
* urutan aktivitas;
* level kesulitan;
* struktur TP.

### 3.4 Prinsip Konsistensi
Apabila satu istilah diperbaiki,
maka seluruh penggunaan istilah yang sama
wajib diperiksa.

### 3.5 Prinsip Beban Kognitif Minimum
Setiap revisi harus:
* mengurangi kebutuhan membaca ulang;
* mengurangi kebutuhan interpretasi;
* mengurangi kebutuhan mengambil keputusan;
* mengurangi kebutuhan mengingat.

## BAB 4 — SUMBER KEPUTUSAN
Urutan keputusan editor:
1. Audit OS
2. Workflow
3. Editor OS

Editor tidak boleh mengubah keputusan auditor.

## BAB 5 — RUANG LINGKUP
Editor boleh memperbaiki:
* redaksi;
* urutan;
* UCAP;
* aksi;
* cue;
* diferensiasi;

selama perubahan tersebut berasal dari Audit OS.

## BAB 6 — LARANGAN
Editor tidak boleh:
* membuat file perantara;
* melakukan audit ulang;
* mencari temuan baru;
* mengubah desain pembelajaran;
* menambah atau menghapus aktivitas;
* menambah variasi bahasa;
* melakukan optimasi di luar Audit OS.

## BAB 7 — KONSISTENSI
Satu temuan menghasilkan satu perbaikan.
Satu perbaikan tidak boleh menghasilkan
perubahan yang tidak berkaitan.

## BAB 8 — KRITERIA EDIT YANG BAIK
Edit yang baik memiliki ciri:
* seluruh temuan diterapkan;
* tidak ada perubahan tambahan;
* tidak mengubah tujuan pedagogis;
* tidak meningkatkan beban kognitif;
* tidak menimbulkan temuan baru.

## BAB 9 — HASIL EDIT
Hasil edit adalah satu file tp-xx-v6.js yang:
* siap live di browser;
* mempertahankan struktur asli;
* mempertahankan tujuan pembelajaran;
* telah menerapkan seluruh rekomendasi audit.

## BAB 10 — PRINSIP NORMATIF
Editor bukan penulis ulang TP.
Editor adalah pelaksana keputusan Audit OS.

Keberhasilan editor diukur dari kemampuan
menghasilkan satu file tp-xx-v6.js yang:
* seminimal mungkin berubah;
* semaksimal mungkin sesuai Audit OS;
* siap live di browser.
